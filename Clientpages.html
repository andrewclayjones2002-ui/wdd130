Client Sites

General Experience and Reflections

The most challenging part of this project was definitely the layout, especially trying to get the rules and images for the Hand & Foot site to stay centered and look right on every screen size. I was able to overcome this by diving deep and learning more about CSS to make sure everything stayed readable and accessible for users. I really enjoyed having different roles, like being the lead dev for the Mcwilliam site where I focused on a clean homepage, but also working as a junior dev on the Cheney and Alvey projects working on child pages. If I did this again, I’d fix my file paths much earlier to avoid those broken local host links and maybe try using a CSS Grid to make my portfolio look even more professional. Overall, I feel way more confident now because I’ve been able to implement what I learned in the semester in these webpages.

Mcwilliam egg website

Being the lead dev wasn’t as hard as I though it would be. To communicate with my team, I made a group chat and communicated task and changes I needed done. I mainly communicated with my client in class and he ended up really liking his website when we published it to him. Github was dieffiently a learning experience, but I learned how it worked quickly and had no problem sharing it with my team and using it. This site was about how to make eggs.
